angular常用的内置指令

angular常用的内置指令

  • angular有内部指令和自定义指令
  • 内置指令 凡是以ng-或者data-ng-为前缀的都是angularJS的内置指令
    指令是放在DOM元素上用来操作DOM元素的方法
  • 自定义指令就不要以ng-或者data-ng-为前缀了 以后的文章中我们还会写到自定义指令
    下面文章总结的就是一些angular常用的内置指令

ng-app

  • 是angularJS读取页面的起始位置
  • 会形成$rootscope的跟作用域
  • 只有最外层的ng-app是有效的 而且只能有一个跟作用域
  • 可以写在任何一个位置 我们写在哪里哪一部分就归angularJS管理
  • 我们一般让一整个页面都归angularJS管理 所以我们的ng-app都写在HTML标签中

ng-init

初始化数据用的，在angularJS中初始化一个数据就好比我们在页面中直接写入一个数据，ng-init用在标签中 并且通过{ { } }来展示到页面当中，要说明的是ng-init一般都是我们用在dome中用来更方便演示效果用的，直接用ng-init初始化数据更加方便，在真实的项目中是几乎不会出现的。

{ { } }

说到ng-init我们初始化数据我们用{ { } }来解析和绑定数据
{ { } }也是angularJS一个重要的特性
在{ { } }我们可以放置文字，变量（变量中可以存储字符串、数组、对象…），运算符（加减乘除 三元运算符），表达式

ng-model

  • ng-model使用在input select textarea和自定义的文本框中（有使用环境的特殊性）
  • ng-model定义的属性名存储的是文本框中的value值
  • 定义的属性value存放在当前作用域下的数据模型上
  • 用{ {value} }取出数据模型上的value对应的值绑定在页面上
  • 最主要额特性是用ng-model实现了数据的双向绑定

ng-repeat (迭代)

  • ng-repeat 是用来重复一个HTML元素的 实现了多数据的绑定 首先循环遍历我们要绑定的数据 并且把没一项数据都绑定到我们制定的DOM元素上从而事项了多数据的绑定 在使用的时候我们想让哪一个标签创建新的DOM元素我们就把ng-repeat加在那个标签上
  • 可以迭代的内容有字符串 数组 对象
  • 对象 a in name(要迭代的数据) 其中的a为value值 对象(key,value) in name(要迭代的数据) 其中的key , value 代表的是一个键值对应的属性名和属性值
  • 数组和字符串中 (key,value) in name(要迭代的数据) 其中的 key , value 代表的是对应的索引和对应的值
  • 当页面中的数据有重复项的时候在加载页面的时候会出现报错的情况，因为在ng-repeat中我们的数据是不可重复的，解决这样的问题我们需要用在追加 track by $index 来解决报错的问题
  • { {$first} } { {$last} } { {$odd} } { {$even} } 这样的都是布尔类型的值 返回出来的为 true false
  • { {$first} } 判断当前项是否为第一项
  • { {$last} } 判断当前项是否为最后一项
  • { {$odd} } 判断当前项是否为奇数项
  • { {$even} } 判断当前项是否为偶数项
  • { {$index} } 表示的是当前项的索引
  • 形成的每一个DOM元素都会有一个自己的私有作用域 这是ng-repeat的一个特性

ng-repeat嵌套

ng-repeat是可以嵌套使用的，我们在外面的标签上使用ng-repeat可以创建DOM元素在内层中同样还是可以使用ng-repeat的来进一步创建我们需要的DOM元素并且绑定数据的

ng-class

  • 动态绑定样式名的
  • ng-class=’’ 可以跟一个值，也可以跟一个{ }用来绑定对个样式名

ng-style

  • 增加动态的行内样式

隔行变色

用以上的内置指令我们就可以实现一个简单的效果了 我们经常用的隔行变色 需要用到ng-init在小demo中来初始化一下我们的数据还需要用到我们的ng-repeat来实现我们的创建DOM元素 用ng-class来给我们的DOM元素添加样式名 { { } }来实现我们的数据的绑定到页面中 { {$odd} }和{ {$even} }来取到我们的奇偶行

ng-if

  • 操作我们的DOM元素是否存在的
  • ng-if=’true’ 当值为true的时候DOM元素为存在 为false的时候是不存在

ng-show 和 ng-hide

  • 操作DOM元素显示和隐藏的 其原理是通过css样式来操作元素的
  • ng-show=’true’ 当为true的时候DOM元素是显示的状态 为false的时候是隐藏的状态
  • ng-hide和ng-show正好相反 当值为true的时候 DOM元素是隐藏的 当值为false的时候DOM元素是显示的

ng-include

  • 用来引入HTML片段的 在不同的页面结构中 有的时候会有相同HTML结构 那么我们就把向同的HTML结构写在一个片段里面 当我们需要的时候我们通过 ng-include 指令来引入我们的 HTML片段达到我们共同代码一起使用的目的这样不仅节省了空间还节省了代码
  • 在ng-include=’’片段名称’’ 里面我们用的是片段名称的字符串

ng-bind 和 ng-bind-template 和 ng-cloak

  • 以上者三个指令都是用来解决我们在绑定数据的时候的闪烁问题的 直接通过{ { } }绑定我们的数据到页面的时候在刷新页面的时候会出现闪烁的问题 这时候我们就要用到这三个方法来解决问题了
  • ng-bind其实{ { } }就是ng-bind指令的缩写 但是直接用ng-bind方法就不会出现闪烁问题
  • ng-bind-template指令可以绑定多个数据实现多数据的同时绑定
  • ng-cloak的指令需要一个隐藏样式 这个指令是真实项目中常用的
